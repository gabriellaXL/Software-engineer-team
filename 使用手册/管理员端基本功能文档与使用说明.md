## 1. 文档说明

本文档用于说明“学院学生综合服务与党团管理平台”管理员端的访问方式、主要页面、已实现能力、使用步骤和当前版本限制，方便互测同学和项目组成员进行管理员端测试。

### 1.2 编写口径

本文档按“实事求是”原则编写：

- 已写入正文的功能，均已从前端页面和后端接口代码中确认存在。
- 需要真实浏览器、线上环境、测试数据或文件上传配合验证的内容，会明确标注为“待人工确认”。

## 3. 后台首页

### 3.1 功能说明

后台首页是管理员端的总览页面，用于集中查看当前系统中的待处理事项和基础数据概况。

### 3.2 页面内容

代码层面确认后台首页展示以下信息：

- 待处理事项数量。
- 模板数量。
- 知识库条目数量。
- 学生数量。
- 待审批事项列表。
- 近期通知。
- 近期申请动态。

### 3.3 操作步骤

1. 使用管理员账号登录系统。
2. 进入“后台首页”。
3. 查看顶部统计卡片。
4. 查看“审批队列”“近期通知”“系统使用情况”区域。
5. 可点击“进入审批”进入审批管理页面。
6. 可点击“管理”进入通知管理页面。

### 3.4 当前版本说明

后台首页的数据依赖后端接口和数据库中的实际数据。如果当前数据库中没有申请、通知、模板或用户数据，对应区域可能显示 `0`、空列表或“暂无”提示。

### 3.5 待人工确认

- 顶部统计数字是否正常显示。
- “进入审批”和“管理”按钮是否能跳转到对应页面。

## 4. 用户管理

### 4.1 功能说明

用户管理用于维护系统账号。当前代码层面确认支持查看用户列表，并提供新建、编辑、删除用户的页面表单和后端接口。

### 4.2 页面字段

用户列表包含以下信息：

- 账号
- 姓名
- 角色
- 专业
- 年级
- 电话
- 邮箱
- 状态
- 操作

用户表单包含以下字段：

- 账号
- 初始密码或重置密码
- 姓名
- 角色
- 状态
- 电话
- 邮箱
- 专业
- 年级
- 部门

### 4.3 操作步骤

查看用户：

1. 登录管理员端。
2. 点击左侧“用户管理”。
3. 查看用户列表。

新建用户：

1. 在右侧“账号信息”表单中填写账号、初始密码、姓名、角色等信息。
2. 点击“创建用户”。
3. 查看用户列表是否刷新。

编辑用户：

1. 在用户列表中点击“编辑”。
2. 系统会将该用户信息载入右侧表单。
3. 修改信息后点击“保存用户”。

删除用户：

1. 在用户列表中点击“删除”。
2. 系统弹出确认提示。
3. 确认后删除用户。

### 4.4 当前版本说明

- 前端对管理员、老师、学院领导等管理类账号的编辑有限制，管理员账号尝试编辑这些账号时可能提示“无权修改管理员/老师/学院领导的账号信息”。
- 删除操作是破坏性操作，会连带删除该用户关联的学生档案、申请、成绩分析、流程提交等数据。测试时不要删除真实管理员账号或项目组正在使用的账号。
- 页面中的表格搜索框目前只建议作为页面展示理解，是否完成真实筛选需实际确认。

### 4.5 待人工确认

请实际确认：

- 用户列表是否能正常加载。
- 新建一个测试学生账号是否成功。
- 编辑测试学生账号信息是否成功。
- 删除刚创建的测试账号是否成功。
- 不要测试删除 `admin001`。

## 5. 知识库管理

### 5.1 功能说明

知识库管理用于维护智能咨询中的政策条目、关键词、标准答复和附件。当前代码层面确认支持查看条目、编辑已有条目、新增条目，并支持上传附件。

### 5.2 页面内容

列表字段包括：

- 标题
- 分类
- 附件
- 状态
- 更新日期
- 操作

表单字段包括：

- 标题
- 分类
- 关键词
- 内容
- 附件上传
- 状态

### 5.3 操作步骤

新增知识库条目：

1. 点击左侧“知识库管理”。
2. 在右侧表单填写标题、分类、关键词、内容。
3. 如需上传附件，选择附件文件。
4. 选择状态为“启用”或“草稿”。
5. 点击“保存条目”。

编辑知识库条目：

1. 在列表中点击目标条目的“编辑”。
2. 页面将内容载入右侧表单。
3. 修改后点击“保存修改”。

### 5.4 当前版本说明

- 附件上传大小限制为 30MB。
- 后端会将知识库附件保存到 `/uploads/policies/` 下，并在条目中记录附件地址。
- 知识库条目保存后会同步生成或更新问答数据，供智能咨询检索使用。
- 页面顶部的“批量导入”按钮当前未从代码中确认完整导入流程，暂不写作可用功能。

### 5.5 待人工确认

请实际确认：

- 新增一条测试知识库条目后，列表是否刷新。
- 编辑该条目后，内容是否保存。
- 学生端智能咨询能否搜到该条目。
- 附件上传后能否下载。
- “草稿”状态条目是否会影响学生端检索结果。

## 6. 通知管理

### 6.1 功能说明

通知管理用于管理员发布系统通知。当前代码层面确认支持填写通知标题、简介、正文，选择标签和推送年级，上传附件，保存本地草稿，预览通知，发布通知和删除通知。

### 6.2 页面内容

通知录入区包含：

- 通知标题
- 通知简介
- 通知内容
- 标签设置
- 自定义标签
- 推送年级
- 附件上传

右侧区域包含：

- 推送范围摘要
- 附件与简介说明
- 草稿箱

下方区域包含：

- 已发布通知列表
- 删除通知按钮

### 6.3 操作步骤

发布通知：

1. 点击左侧“通知管理”。
2. 填写通知标题、简介、正文。
3. 选择标签和推送年级。
4. 如需附件，上传通知附件。
5. 点击“确认发布”或“保存并发布”。
6. 查看下方“已发布通知”列表是否新增。

保存草稿：

1. 填写通知内容。
2. 点击“存为草稿”。
3. 在草稿箱中查看草稿。

预览通知：

1. 填写通知内容。
2. 点击“预览通知”。
3. 查看预览弹窗或页面提示。

删除通知：

1. 在“已发布通知”列表中点击“删除”。
2. 确认删除。

### 6.4 当前版本说明

- 发布通知会调用后端通知接口，成功后通知应写入数据库。
- 草稿箱使用浏览器本地 `localStorage` 保存，不代表已经提交到后端。
- 通知附件以数据形式随通知保存，学生端能否下载需结合实际浏览器和数据确认。
- 推送年级用于限制学生端可见范围，实际效果需要用学生账号验证。

### 6.5 待人工确认

请实际确认：

- 发布通知后管理员端列表是否新增。
- 使用学生账号登录后，目标年级学生是否能看到该通知。
- 非目标年级学生是否看不到该通知。
- 通知附件能否在学生端详情页下载。
- 删除通知后学生端是否不再显示。

## 7. 流程配置

### 7.1 功能说明

流程配置用于维护入党、入团流程节点，并查看党团流程材料的待审核记录。当前代码层面确认支持流程节点的查看、新增、编辑、删除，以及待审流程材料的审核入口。

### 7.2 页面内容

页面包含：

- 入党流程 / 入团流程切换。
- 节点列表。
- 节点参数表单。
- 流程审核列表。

节点字段包括：

- 流程类型
- 顺序
- 节点名称
- 详细说明
- 开始时间
- 截止时间
- 提醒规则
- 节点附件

### 7.3 操作步骤

查看流程节点：

1. 点击左侧“流程配置”。
2. 在顶部切换“入党流程”或“入团流程”。
3. 查看节点列表。

新增节点：

1. 在右侧表单填写流程类型、顺序、节点名称等内容。
2. 如需附件，上传节点附件。
3. 点击“创建节点”。

编辑节点：

1. 在节点列表中点击“编辑”。
2. 修改节点内容。
3. 点击“保存节点”。

删除节点：

1. 在节点列表中点击“删除”。
2. 确认删除。

审核流程材料：

1. 在“流程审核”区域查看待审核材料。
2. 点击“审核”。
3. 填写审核意见。
4. 点击“通过”或“退回补充”。

### 7.4 当前版本说明

- 删除流程节点会影响该节点下的流程记录，测试时需谨慎。
- 流程材料审核依赖学生端实际提交材料。没有学生提交时，列表会显示暂无待审核材料。
- 节点附件和学生提交附件能否下载，需要实际浏览器确认。

### 7.5 待人工确认

请实际确认：

- 入党/入团流程切换是否正常。
- 新增一个测试节点是否成功。
- 编辑测试节点是否成功。
- 删除测试节点是否成功。
- 学生端能否看到管理员新增或修改的流程节点。
- 学生提交材料后，管理员端是否出现待审核记录。
- 审核通过或退回后，学生端状态是否变化。

## 8. 审批管理

### 8.1 功能说明

审批管理用于集中查看和处理学生证明申请与党团流程材料。当前代码层面确认支持查看申请列表、进入申请详情、通过或退回证明申请，并支持查看党团流程待审核材料。

### 8.2 页面内容

审批管理包含两个区域：

- 申请列表：展示证明申请记录。
- 流程材料：展示党团流程待审核材料。

申请详情页包含：

- 申请人
- 申请单号
- 当前状态
- 用途
- 说明
- 补充附件下载入口
- PDF 预览或下载入口
- 审批意见
- 通过 / 退回补充按钮

### 8.3 操作步骤

审核证明申请：

1. 点击左侧“审批管理”。
2. 在申请列表中点击“查看详情”。
3. 阅读申请信息和附件。
4. 填写审批意见。
5. 点击“通过”或“退回补充”。
6. 返回列表查看状态变化。

审核党团流程材料：

1. 在“流程材料”区域查看待审核材料。
2. 点击“查看详情”。
3. 填写审核意见。
4. 点击“通过”或“退回补充”。

### 8.4 当前版本说明

- 证明申请审核会调用后端 `/api/applications/review` 接口，审核结果会更新申请状态。
- 党团流程材料审核会调用 `/api/process/submissions/:id/review` 接口。
- PDF 预览和附件下载是否可用，取决于学生提交数据中是否包含 `fileData` 或附件数据。
- 如果当前没有学生提交的申请或材料，页面会显示暂无记录。

### 8.5 待人工确认

请实际确认：

- 学生提交证明申请后，管理员端是否能看到该申请。
- 管理员点击“通过”后，学生端是否显示“已通过”。
- 管理员点击“退回补充”后，学生端是否显示“待补充”。
- PDF 预览或下载是否正常。
- 补充附件下载是否正常。
- 党团流程材料审核后，学生端流程状态是否变化。

5. 点击“上传模板”。

## 10. 培养方案

### 10.1 功能说明

培养方案页面用于维护专业培养方案、适用年级、状态和附件。当前代码层面确认支持查看培养方案列表，新建、编辑、删除培养方案，并支持上传培养方案文件。

### 10.2 页面内容

列表字段包括：

- 方案名称
- 适用年级
- 状态
- 培养方案文件
- 最近更新
- 操作

表单字段包括：

- 方案名称
- 适用年级
- 专业方向
- 状态
- 培养方案文件

### 10.3 操作步骤

新建培养方案：

1. 点击左侧“培养方案”。
2. 在右侧表单填写方案名称、适用年级、专业方向。
3. 选择状态：草稿、待审核或已发布。
4. 如需附件，上传培养方案文件。
5. 点击“创建方案”。

编辑培养方案：

3. 点击“保存方案”。

删除培养方案：

### 10.5 待人工确认
