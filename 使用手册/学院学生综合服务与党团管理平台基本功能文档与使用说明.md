## 1. 文档说明

### 1.1 编写目的
本文档用于说明“学院学生综合服务与党团管理平台”的系统访问方式、测试账号、核心功能、操作流程、示例输入输出及已知限制，供测试小组、学生用户和管理员用户在互测与演示阶段参考使用。

### 1.2 适用对象
本文档适用于本系统的测试人员、学生用户和管理员用户。

### 1.3 文档范围
本文档主要包含以下内容：

- 系统访问方式与运行说明
- 测试账号与角色说明
- 学生端核心功能说明
- 管理员端核心功能说明
- 常见问题与已知限制

## 2. 系统简介

“学院学生综合服务与党团管理平台”面向学生与管理员提供统一的事务管理服务。学生用户可通过本系统完成通知查看、智能咨询、党团流程查看、相关业务申请提交、申请进度查询、模板下载及成绩分析等操作；管理员用户可通过后台完成用户管理、通知管理、知识库维护、审批处理、模板管理和培养方案管理等操作。

本系统旨在提升学生事务办理效率，减少信息分散带来的重复沟通成本，为学生提供更便捷的线上服务入口。

## 3. 使用环境与登录方式

### 3.1 使用环境
建议使用以下浏览器访问系统：

- Google Chrome
- Microsoft Edge
- Safari
- 其他支持现代 Web 标准的浏览器

### 3.2 系统访问地址
系统访问地址如下：

- 系统访问地址：[`http://10.10.0.24:3000/`](http://10.10.0.24:3000/)

### 3.3 项目启动方式
当前测试环境以已部署版本为主，测试人员可直接通过上述地址访问系统。

如需本地或新环境启动项目，可参考仓库中的后端启动说明和单机部署说明，基本流程如下：

1. 准备数据库环境并完成建库。
2. 在 `backend` 目录配置 `.env`。
3. 安装后端依赖。
4. 执行数据库初始化脚本。
5. 启动后端服务。
6. 通过浏览器访问系统页面。

### 3.4 测试账号
如使用测试环境，可填写测试账号：

- 学生账号：`stu001`
- 密码：`123456`
- 管理员账号：`admin001`
- 管理员密码：`123456`

说明：本文档中的操作截图基于系统测试环境账号生成，实际使用时请以管理员分配的账号信息为准。

### 3.5 登录步骤
1. 打开系统访问地址。
2. 在登录页面输入对应角色的账号和密码。
3. 点击“登录”按钮。
4. 登录成功后进入对应角色首页。

### 3.6 登录说明
- 若账号或密码输入错误，系统会给出提示信息。
- 若页面无法正常加载，请检查网络连接或联系系统维护人员。
- 若登录成功后页面空白或功能异常，建议刷新页面后重试。

### 3.7 角色与核心功能概览

| 角色 | 主要功能 |
|------|----------|
| 学生 | 智能咨询、通知公告、党团流程、证明申请、模板下载、成绩分析、申请记录查询 |
| 管理员 | 后台首页、用户管理、知识库管理、通知管理、审批管理、模板管理、培养方案管理 |

### 3.8 示例输入输出

- 智能咨询示例输入：`在读证明`
- 智能咨询预期输出：返回与“在读证明”相关的知识条目、说明信息及模板推荐
- 学生申请示例输入：填写申请类型、用途说明并上传材料
- 学生申请预期输出：系统提示提交成功，并在申请记录中显示一条新的申请记录

### 3.9 已知说明与限制

1. 本文档中的截图基于测试环境账号生成，实际账号信息以管理员分配结果为准。
2. 当前系统部分页面内容依赖初始化测试数据展示，若数据为空，页面可能只显示空列表或默认提示。
3. 智能咨询页中的部分模板下载入口为演示交互，现阶段主要用于展示“知识条目关联模板资源”的产品形态，不强制要求在当前测试版本中实现完整真实文件下载，具体下载能力以实际部署版本为准。
4. 若测试环境未准备成绩单文件或部分角色测试数据，相关功能可能无法完整展示全部效果。
5. 本文档已对当前版本中的演示交互、测试数据依赖和功能限制进行明确说明。互测时应优先按本文档描述理解系统行为；但若页面提示、按钮文案与本文档说明明显不一致，仍可能被判定为逻辑缺陷或文档缺陷。
6. 党团流程页面当前主要完成了节点可视化展示、状态区分、历史记录展示和表单交互提示；“保存草稿”“提交审核”在当前测试版本中主要体现为前端提示反馈，不代表已完成独立草稿箱或完整后端持久化。

## 4.4 智能咨询

### 4.4.1 功能说明
学生可在本模块输入关键词，查询与学生事务相关的政策说明、常见问题和业务办理指引。页面通常展示搜索框、分类筛选、知识库条目列表和标准答复区域，便于学生快速定位材料要求与办理说明。

### 4.4.2 操作步骤
1. 点击“智能咨询”模块。
2. 在搜索框中输入关键词，如“在读证明”“奖学金”“党员发展”等。
3. 根据需要选择分类标签，如“全部”“证明类”“党团类”“奖助类”“生活类”。
4. 点击查询按钮。
5. 查看系统返回结果。
6. 在右侧查看标准答复、附件名称以及页面提供的“下载”“官方渠道”等入口提示。

### 4.4.3 预期结果
- 系统返回与关键词相关的政策说明或知识条目。
- 查询结果应包含标题、分类、更新时间、维护部门、附件名称或答案摘要等信息。
- 页面右侧可展示标准答复区域，用于汇总当前命中条目的说明信息。

### 4.4.4 注意事项
- 建议先测试常见关键词，确认系统基础查询能力。
- 可结合分类标签一起使用，以缩小检索范围并观察不同类别下的返回结果。
- 当前版本中，智能咨询结果区域的部分模板下载入口为演示交互，可用于展示功能流程与模板关联能力；若点击后未触发真实文件下载，可按已知限制理解，不作为当前阶段必须完成的真实下载闭环。
- “官方渠道”按钮当前同样为演示交互。若页面提示“已复制官方渠道说明编号”，但未出现真实剪贴板内容或未跳转外部链接，应按当前版本的演示说明理解。
- 若后续页面文案调整为“模拟下载”“渠道说明”等更明确表述，应以实际部署版本为准，并同步更新本文档。
- 也可补充一个“无结果关键词”的测试场景，观察系统提示是否合理。

## 4.5 查看党团流程

### 4.5.1 功能说明
学生可在本模块查看党团流程节点、当前所处状态、办理要求及提醒信息。页面通常包含节点时间线、节点操作区和历史记录区，可用于展示当前培养阶段、材料类型选择、说明填写以及审批日志。

### 4.5.2 操作步骤
1. 点击“党团流程”模块。
2. 查看流程时间线及各节点状态。
3. 阅读当前节点说明及相关提示。
4. 在右侧“节点操作”区域选择材料类型。
5. 在说明框中填写补充说明，并按需选择附件文件。
6. 点击“保存草稿”或“提交审核”按钮，观察系统提示信息。
7. 查看下方“审批与操作日志”区域是否展示历史记录。

### 4.5.3 预期结果
- 页面正常显示流程节点信息。
- 用户可看到当前状态、已完成节点及后续提醒。
- 页面右侧显示节点操作区，包括材料类型、说明输入框和附件上传控件。
- 页面下方显示历史记录表格，包括时间、节点、操作、处理人和结果。

### 4.5.4 注意事项
- 若流程节点数据不完整，需确认数据库中是否有对应测试记录。
- 若当前账号无相关流程数据，可在备注中说明“当前账号暂无流程数据”。
- 当前版本中，“保存草稿”和“提交审核”按钮主要用于展示节点操作流程。根据现有实现，点击“保存草稿”会提示“草稿已保存”，点击“提交审核”会提示“内容已保存”，但前端未看到独立草稿箱入口，后端也未提供对应的草稿保存接口。
- 因此，现阶段测试应重点关注页面展示、节点状态、表单区域和提示反馈是否正常，不应将该处理解为已完整实现了真实的草稿持久化或材料提交流程。
- 若后续版本补充了真实的材料上传、草稿保存或审核流转能力，应以实际系统行为和后端接口为准，并同步更新本说明。

## 5. 管理员端基本功能说明

### 5.1 登录系统

#### 5.1.1 功能说明
管理员通过后台账号登录系统，登录成功后进入管理员端后台首页。

#### 5.1.2 操作步骤
1. 打开系统访问地址。
2. 输入管理员账号和密码。
3. 点击“登录”按钮。
4. 确认系统进入管理员后台首页。

#### 5.1.3 预期结果
- 系统成功进入管理员后台。
- 页面显示管理员可使用的功能菜单。

#### 5.1.4 注意事项
- 若管理员账号无法登录，请检查账号状态和密码配置。
- 若页面样式异常或无数据，建议刷新页面后重试。

## 6. 常见问题

### 5.1 登录失败怎么办
请检查账号、密码和系统访问地址是否正确。若仍无法登录，请联系管理员确认账号状态。

### 5.2 页面无数据怎么办
请确认当前系统是否已初始化测试数据，同时检查后端服务与数据库连接是否正常。

### 5.3 上传失败怎么办
请检查上传文件的格式、大小及网络连接情况，必要时更换浏览器后重试。

### 5.4 提交申请后看不到记录怎么办
建议刷新页面后重新查看。如仍无数据，请记录操作步骤和时间并反馈维护人员排查。

### 5.5 页面报错怎么办
请记录报错信息、出错页面、触发步骤及时间，便于后续定位问题。
